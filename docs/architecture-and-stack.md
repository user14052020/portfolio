# Architecture And Stack Decision

## Рекомендация

Для этого проекта `ComfyUI` подходит, но только как слой генерации медиа.

Рекомендуемая схема:

- `ComfyUI`:
  - генерация изображений по prompt;
  - generation from image/reference;
  - later: image-to-video / text-to-video, если это реально нужно продукту;
  - хранение и исполнение workflow.

- `LLM service`:
  - анализ текста пользователя;
  - определение тона ответа;
  - рекомендация, что с чем надеть;
  - юмористический / саркастичный / циничный стиль, но в контролируемых рамках;
  - классификация пользовательского запроса:
    - только совет;
    - совет + генерация;
    - совет + подбор из каталога;
    - отказ / safe fallback.

- `Catalog / retrieval layer`:
  - поиск и фильтрация вещей из локальных секондов;
  - дедупликация;
  - ранжирование по релевантности образу;
  - выдача пользователю ссылок и карточек товаров.

## Почему ComfyUI стоит оставить

ComfyUI хорош именно там, где нужны:

- локальный запуск моделей;
- тонкий контроль workflow;
- image-to-image / reference-to-image пайплайны;
- возможность позже нарастить video workflow;
- кастомные ноды и API-ориентированная интеграция.

Для этого проекта это совпадает с задачей генерации "примеров вещей, которые дополнят образ".

## Где ComfyUI не должен быть главным

ComfyUI не должен отвечать за:

- характер и тон стилиста;
- бизнес-логику чата;
- выбор, когда именно нужно генерировать картинку;
- поиск и ранжирование реальных товаров;
- память диалога и профиль пользователя.

Если пытаться делать всё через один workflow в ComfyUI, проект станет хрупким и тяжело поддерживаемым.

## Что лучше для текста

### Выбранный вариант: vLLM

Для вашего текущего плана выбран `vLLM`.

Почему:

- нужен OpenAI-compatible API;
- нужна нормальная server-side подача модели, а не только локальный single-user режим;
- нужно строить backend integration вокруг отдельного inference server;
- позже можно масштабировать throughput и менять модели без переделки логики приложения.

Использовать для:

- стилистических рекомендаций;
- tone control;
- route decision: нужен ли рендер, нужен ли каталог, нужен ли просто совет;
- structured JSON output для catalog matching.

## А есть ли альтернатива ComfyUI

### InvokeAI

Не "лучше" в абсолюте, а лучше только в другом сценарии.

Подойдёт, если фокус смещается в сторону:

- artist-friendly UI;
- ручной creative workflow;
- меньшей потребности в сложной node orchestration;
- более продуктового image studio UX.

Но для вашего кейса, где уже есть локальные модели и нужен backend-driven workflow, ComfyUI выглядит практичнее.

## Итоговое архитектурное решение

Оставляем:

- `ComfyUI` как media generation engine.

Добавляем:

- `vLLM` как text brain;
- отдельный parser/catalog pipeline;
- отдельный ranking layer между LLM и catalog.

Не делаем:

- "весь AI на одном ComfyUI";
- прямой парсинг сайтов из chat request;
- блокирующую генерацию до того, как пользователю уже отдан текстовый ответ.

## Целевой поток запроса

1. Пользователь отправляет текст или изображение.
2. Backend сохраняет запрос и запускает text pipeline.
3. LLM определяет:
   - intent;
   - tone;
   - нужно ли генерировать картинку;
   - нужно ли искать реальные вещи.
4. Пользователь сразу получает текстовый ответ стилиста.
5. Если нужен render:
   - backend ставит generation job;
   - ComfyUI исполняет workflow;
   - результат появляется в чате асинхронно.
6. Если нужен поиск по каталогу:
   - retrieval layer подбирает реальные товары;
   - бот показывает карточки вещей и поясняет, почему они подходят.

## Что важно для тона бота

Нужно сразу разделить:

- `brand voice`;
- `allowed humor`;
- `forbidden behavior`.

Рекомендуемый тон:

- остроумный;
- уверенный;
- местами ироничный;
- не унижающий пользователя;
- не токсичный;
- не переходящий в оскорбления внешности, возраста, веса, пола или социального статуса.

Это нужно оформить отдельным prompt policy и тест-кейсами.

## Моя рекомендация

Вывод по текущему состоянию проекта:

- `ComfyUI` менять не надо;
- для текста используем `vLLM`;
- в первый этап не лезть глубоко в video generation;
- сначала сделать сильный image generation + catalog matching;
- parser и retrieval должны быть отдельным контуром, а не частью workflow в ComfyUI.
