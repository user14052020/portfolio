# AI Stylist Docs

Эта папка фиксирует, что делаем дальше по проекту AI-стилиста и в каком порядке.

## Что читать сначала

1. [architecture-and-stack.md](./architecture-and-stack.md)  
   Решение по стеку: что оставляем на ComfyUI, что выносим в отдельный LLM-сервис, какие есть альтернативы.

2. [server-setup-checklist.md](./server-setup-checklist.md)  
   Практический checklist по настройке сервера, сервисов, сетей, очередей и мониторинга.

3. [product-roadmap.md](./product-roadmap.md)  
   Этапы развития продукта: от текущего MVP к полноценному AI-стилисту с каталогом вещей.

4. [parser-and-catalog-plan.md](./parser-and-catalog-plan.md)  
   План по парсеру локальных секондов, нормализации данных и выдаче подходящих товаров пользователю.

## Короткий вывод

- ComfyUI оставляем как движок генерации изображений и видео.
- Текстовые рекомендации, тон общения, юмор и ранжирование вещей не вешаем на ComfyUI.
- Для текста используем `vLLM` как отдельный LLM-сервис.
- Парсер делаем как отдельный ingestion pipeline, а не как часть чата.

## Порядок выполнения

1. Зафиксировать архитектуру и границы сервисов.
2. Довести серверную инфраструктуру до предсказуемого состояния.
3. Добавить реальный текстовый мозг стилиста.
4. Стабилизировать пайплайн генерации образов.
5. Подключить каталог вещей и парсер источников.
6. Добавить ранжирование, персонализацию и аналитику.
