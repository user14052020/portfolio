# Product Roadmap

## Видение

Сделать AI-стилиста, который:

- принимает текст и изображения от пользователя;
- отвечает в фирменном тоне;
- объясняет, почему вещь удачная или неудачная;
- предлагает, с чем её носить;
- при необходимости генерирует референс-изображение образа;
- предлагает реальные вещи из локальных секондов;
- постепенно учится лучше подбирать вещи.

## Этап 1. Стабильный MVP стилиста

Цель:

- быстрый чат;
- понятные текстовые рекомендации;
- асинхронная генерация референса.

Задачи:

- [ ] Довести UX чата.
- [ ] Перевести стилистические ответы с mock logic на реальный LLM.
- [ ] Зафиксировать 1 production workflow генерации образа.
- [ ] Добавить логику: когда генерировать картинку, а когда нет.
- [ ] Ввести prompt policy и tone policy.

Результат:

- пользователь получает внятный текстовый совет;
- иногда получает картинку как "пример образа";
- система не выглядит случайным генератором картинок.

## Этап 2. Personality и tone control

Цель:

- бот должен звучать живо и узнаваемо.

Задачи:

- [ ] Описать 2-3 допустимых режима тона:
  - friendly;
  - sharp;
  - sarcastic.
- [ ] Ввести safety-ограничения.
- [ ] Написать тестовый набор user prompts.
- [ ] Проверить, что бот не уходит в токсичность и не хамит без повода.

Результат:

- у бота есть характер;
- tone управляемый, а не хаотичный.

## Этап 3. Outfit intelligence

Цель:

- бот реально понимает, что с чем носить.

Задачи:

- [ ] Сформировать структуру outfit recommendation:
  - main item;
  - complementary items;
  - colors;
  - textures;
  - footwear;
  - accessory;
  - occasion;
  - weather notes.
- [ ] Выделить structured JSON output для backend.
- [ ] Добавить post-processing rules.
- [ ] Сделать ranking кандидатов из каталога по structured output.

Результат:

- совет становится пригодным для автоматического матчинга с реальными товарами.

## Этап 4. Реальные вещи из локальных секондов

Цель:

- помимо красивого AI-референса показывать, что можно реально купить.

Задачи:

- [ ] Поднять parser pipeline.
- [ ] Собрать каталог вещей.
- [ ] Нормализовать категории, размеры, цвета, бренды и condition.
- [ ] Настроить image download/cache.
- [ ] Выводить в чате 3-8 релевантных карточек.

Результат:

- чат перестаёт быть просто "разговором" и начинает вести к реальному действию.

## Этап 5. Персонализация

Цель:

- учитывать вкус и контекст конкретного пользователя.

Задачи:

- [ ] Хранить предпочтения пользователя.
- [ ] Хранить disliked styles / liked styles.
- [ ] Запоминать рост, вес, любимые фасоны, бюджет, размеры.
- [ ] Добавить profile-aware ranking.

Результат:

- рекомендации становятся не общими, а персональными.

## Этап 6. Monetization-ready версия

Цель:

- подготовить проект к реальной эксплуатации.

Задачи:

- [ ] Аналитика воронки.
- [ ] CTR по каталогу.
- [ ] Tracking по генерациям.
- [ ] Moderation and abuse protection.
- [ ] Admin инструменты для источников, моделей и tone presets.

Результат:

- проект можно использовать как реальный сервис, а не только как демо.

## Что не делать прямо сейчас

- Не строить сложный video pipeline до того, как image generation и catalog matching уже работают.
- Не пытаться сразу поддержать десятки источников.
- Не смешивать parser, LLM reasoning и ComfyUI workflow в один процесс.
- Не пытаться "обучать свою модель стилиста" на первом этапе.

## Definition of Done для ближайшего этапа

Ближайший этап завершён, если:

- бот даёт осмысленный текстовый совет;
- бот умеет по решению LLM запускать генерацию образа;
- generation result появляется в чате асинхронно;
- по structured recommendation можно подобрать реальные вещи из каталога;
- хотя бы один источник каталога подключён end-to-end.
